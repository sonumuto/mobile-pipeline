# Food Delivery App

A basic food-delivery app built with **Flutter**, so the same codebase runs on
both **Android** and **iOS**. It's a learning/demo app — generic branding, no
real vendor assets, and no backend (data is mocked).

## Features

- Home screen with a list of restaurants + live search
- Restaurant screen with a menu and add/remove quantity steppers
- A single-restaurant cart (switching restaurants resets it, like most delivery apps)
- Cart screen with price breakdown (subtotal + delivery fee) and a mock "Place order"
- Live cart badge in the app bar

## Project structure

```
lib/
  main.dart                 App entry + cart provider
  theme.dart                Colours and Material theme
  models/models.dart        Restaurant & MenuItem models
  data/mock_data.dart       Demo restaurants and menus
  state/cart_model.dart     Cart logic (ChangeNotifier)
  screens/
    home_screen.dart        Restaurant list + search
    restaurant_screen.dart  Menu + add-to-cart
    cart_screen.dart        Cart + checkout
test/
  cart_model_test.dart      Unit tests for the cart
```

## Running it

You need the Flutter SDK. If it isn't installed:

- macOS: `brew install --cask flutter` (or see https://docs.flutter.dev/get-started/install)

This repo currently only contains the Dart source (`lib/`, `test/`, `pubspec.yaml`).
The native `android/` and `ios/` project folders are generated by Flutter. From
this directory run **once**:

```bash
flutter create .          # scaffolds android/, ios/, etc. around the existing lib/
flutter pub get           # fetch dependencies
```

Then:

```bash
flutter run               # runs on a connected device / simulator
flutter run -d ios        # iOS Simulator (macOS + Xcode required)
flutter run -d android    # Android emulator / device
flutter test              # run the unit tests
```

Build release binaries:

```bash
flutter build apk         # Android APK
flutter build appbundle   # Android Play Store bundle
flutter build ios         # iOS (then archive/sign in Xcode)
```
